# Screenshots

Drop PNGs in this directory with the filenames below; the project [README](../../README.md) references them.

| File | What to show |
|---|---|
| `main-menu.png` | Main menu screen (boot) with the classic green-on-black theme. Cursor on MESSAGE INDEX (Alpine's default). |
| `folder-list.png` | Folder list with the user's real folder tree. A nested folder visible if possible. |
| `index.png` | Message index for the Inbox — several rows including at least one unread (bolded), one flagged (`!`), and one with attachments. |
| `view.png` | View screen on a representative message. Brief headers, body wrapped. |
| `compose.png` | Compose screen mid-edit with content in To / Subject / Body. Cursor visible in the body. |
| `setup.png` | Setup screen showing both rows (Theme + Visual filter) with current values. |

Recommended capture settings:
- 100×30 terminal at default font (xterm.js JetBrains Mono 14 px) — keeps the screenshots a sensible size for README rendering on GitHub.
- macOS: `Cmd+Shift+4` then `Space` then click the Cairn window to capture only the window (no desktop background).
- Linux: `gnome-screenshot -w` or `flameshot gui` with window selection.
- Windows: `Win+Shift+S` window mode.

PNG, not JPEG — text artifacts badly on lossy compression. Lossless WebP would also work if you prefer the smaller files; update the README references accordingly.
